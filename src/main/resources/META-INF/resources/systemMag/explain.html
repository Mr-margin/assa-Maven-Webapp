我想干什么

这是一个基于bootstrap框架的数据管理工具。

在教委、农经办、河北台账、扶贫等项目中，我我们经常遇到数据问题：

  1. 数据量非常大，前期数据处理工作繁琐。
  2. 各种数据关系混乱，没有良好的版本控制，文档更新不及时。
  3. 数据更新滞后，基本上没有形成很好的数据更新机制。
  4. 缺少通用性，每次都是针对项目，没有产品化设计。

此项的目标是通过界面化的管理工具将一些基础的数据管理与维护功能提炼出来，减少前期开发时间。同时探索bootstrap框架下的H+后台啊主题使用方式与设计方式。

技术要点
  1. 前台页面与后台代码分开为两个项目，增加项目可读性。
  2. 前台项目采用bootstrap框架下的H+进行修改开发。
  3. 后台项目框架采用SpringMVC + mybatis。
  4. 数据库暂时采用MySQL。
模块功能说明
基础模块->元数据的结构

所有的数据应该使用元数据进行管理，但是元数据又该有哪些属性呢，这需要根据项目实际情况进行配置，此模块决定了元数据应该有哪些列用于后续管理。

  1. 对应表：metadata_st（元数据结构说明）
  2. 实现增删改功能
  3. 根据内容自动生成相应的sql语句调整元数据表结构
  4. 修改或删除时，元数据表相应列自动清空
基础模块->元数据的内容

元数据表存储的是实际数据的结构，原则上实际项目中所有的表格均由元数据管理。

  1. 对应表：metadata_up（元数据表）metadata_table
  2. 实现增删改功能
  3. 根据内容自动生成相应的sql语句调整数据表结构
  4. 增加识别功能，识别未管理的数据表结构并生成元数据表，需要人工增加备注信息
基础模块->数据表

通过元数据表存储的表名、结构，管理实际数据表以及数据。

  1. 对应表：metadata_up（元数据表）、metadata_table、实际数据表
  2. 实现真实数据的增删改功能
  3. 支持导入导出功能，文件格式为excel
  4. 导入时要求excel文件结构与数据库结构一致
数据处理->生成视图

根据实际数据表生成业务需要的视图，减少前台开发时对数据的关注。

  1. 对应表：metadata_up（元数据表）、metadata_table、实际数据表
  2. 实现真实数据的增删改功能
  3. 支持导入导出功能，文件格式为excel
  4. 导入时要求excel文件结构与数据库结构一致
